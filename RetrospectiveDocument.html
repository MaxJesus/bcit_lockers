Project Overview

  • In a nutshell, what does it do?

The application is for students in BCIT to reservate a locker for their own use, this would promote the use of the lockers, and would make life of students more convenient.

  • Describe MVP in terms of scenarios/tasks/user-stories
    The MVP is actually an imaginary application that requires some hardware upgrades in the current lockers before all the functions are available.

Viewing Vacancies:

The user would not have to login, if they just wanna take a look at the vacancies in different locations. There is a continue without login button for the user to go on. It is a good solution for students who are not able to find available lockers. Within a few clicks, the users would know the current vacancies.

        Booking a locker

To book a locker, the user would need to login with their own account. This is preferred by using myBCIT email address.
The user would follow the guide of the lockers going deeper from the different school sections to the exact floor and pick one specific locker.
There is a Pick one for me function as soon as the user enters the section. By clicking this button, the app will automatically pick a random locker from the section that has the most vacancies, this would try to keep the distribution of the vacancies even.

Locker Management
The management page will allow the user to make digital notes of what is in the locker. By simply dragging items in and out of the locker, the event will interact with the database and keep track of stuff that are in the locker.
The users can create their own item within a few clicks.
Future updates will include reminding functions.

There is an imaginary locking/unlocking function in the page that would allow the users to digitally manipulate the lock(with bluetooth on).

 

Changing a new locker

There is a function that would allow the users to switch to another locker by filling in a form. Customer service is involved.
